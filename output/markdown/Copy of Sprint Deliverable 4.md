## E. ISO/IEC 25010 Software Quality Evaluation

The panel shall evaluate the developed software using the following software quality characteristics.

### Functional Suitability (10 Points)

Minimum requirements:

- Complete implementation of approved project requirements
- Functional CRUD operations where applicable
- Complete business workflow
- Input validation
- Appropriate success and error notifications

### Performance Efficiency (10 Points)

Minimum requirements:

- Acceptable loading time
- Smooth navigation
- Efficient data retrieval
- Optimized resource utilization
- No noticeable performance degradation

### Compatibility (10 Points)

Minimum requirements:

- Mobile-responsive interface
- Tablet compatibility
- Desktop compatibility
- Cross-browser compatibility
- Proper display across different screen resolutions

### Interaction Capability (Usability) (10 Points)

Minimum requirements:

#### Navigation

- Consistent navigation
- Logical workflow
- Easy access to system functions

#### Interface Consistency

- Consistent colors
- Consistent typography
- Consistent buttons
- Consistent icons
- Consistent spacing

#### Data Management

- Search
- Filter
- Pagination or Lazy Loading
- Sorting where applicable

#### Forms

- Required field indicators
- Input validation
- Success messages
- Error messages
- Confirmation dialogs

### Reliability (10 Points)

Minimum requirements:

- Stable system operation
- Proper exception handling
- Data integrity
- Recovery from invalid inputs
- No crashes during demonstration

### Security (10 Points)

Minimum requirements:

#### Authentication

- [x] Secure login authentication
- [x] Password protection
- [x] Role-based access control (where applicable)
- [x] Third-party authentication (e.g., Google, Microsoft, GitHub, Facebook) where applicable

#### Privacy

- [x] Privacy Notice
- [x] Terms and Conditions
- [x] User Consent

#### Data Protection

- [x] Password hashing or secure password storage
- [x] Protection of sensitive information
- [x] Data anonymization or masking where appropriate
- [x] Secure input validation

#### Session Management

- [x] Secure logout functionality
- [x] Prevention of unauthorized access
- [x] Protected routes or pages
- [x] Session timeout or token expiration (where applicable)

### Future Software Quality Characteristics (Not Graded)

The following ISO/IEC 25010 software quality characteristics are acknowledged but are excluded from the current course evaluation due to the scope and duration of the project.

- Maintainability
- Portability

### Performance Descriptors

- **Fully Addressed:** Complete, accurate, professionally organized, and demonstrates excellent software quality with no significant deficiencies.
- **Mostly Addressed:** Minor omissions or inconsistencies are present but do not significantly affect the quality of the software or supporting artifacts.
- **Adequately Addressed:** Major requirements are present, although several improvements are necessary.
- **Partially Addressed:** Significant deficiencies are evident, although the core requirements remain identifiable.
- **Minimally Addressed:** Limited evidence is provided and major requirements remain unmet.
